Notes for review (Rev 2)
  1. Cards are sorted by last updated; the tab badge shows the count (cap 200).
  2. Thumbnail = one mini map per board rendered by the shared BoardThumbnail component (extracted from the Maps-tab previews + guide-page snippets), with hero portraits clipped to hexes and a team-colored ring. In the real app these are the actual character webp portraits; unresolvable units fall back to a team-colored dot.
  3. Select switches the mode picker to the team's mode, loads the snapshot into the boards, overwrites that mode's active team, and makes this team the Save target (see the active-team label in team-mode-controls.html). Toast, no modal.
  4. Select / Duplicate / Delete render at equal width (3-column grid) so actions align across cards.
  5. Delete (per card) and Delete all (header) both use the two-step inline confirm (arms for 3 s). "Replace library" = Delete all + Import.
  6. Duplicate copies the snapshot as "<name> (copy)" (clamped to 60 chars).
  7. Rename is inline (click the name; Enter/blur commits, Esc cancels), max 60 chars.
  8. Cap indicator appears in the header once the library reaches 80% of the 200-team cap.
  9. On mobile this panel lives in the same pull-up sheet as the other roster tabs; selecting a team collapses the sheet so the loaded boards are visible (same pattern as placing a character).
