Mechanics Library — deep dive on Options 2 & 3

Both show all 25 guides, both visual + text. The real question: where does the explanatory text live?

Where does the text live? Every guide is one .vue snippet — a title, the hex diagram, and the prose explaining it — authored once. That single source can be rendered in three places, and the difference between Option 2 and Option 3 is how much of it each surface shows:
  • Hero skill page (today) — the full write-up, in context next to that skill.
  • Option 3 · compendium — the full write-up rendered inline, so you read everything in one place.
  • Option 2 · panel — a teaser (diagram thumbnail + one-line blurb) that links to the full write-up on the hero page.
So there's no content duplication — it's the same component. Option 3 just renders it in full; Option 2 renders a preview and sends you to read it.

The one shared prerequisite (small): snippets currently only render when teleported into a hero page's skill sections (<SkillSnippets> needs the skill-section anchors). To show them anywhere else, we add an inline render mode so the diagram + prose render in place. The 25 .vue files don't change — it's a one-component tweak, and both options need it.
My read: your choices (complete + both visual & text) point at Option 3 — full write-ups make "complete" and "text-only included" cohere, and it's the SEO/shareable win. They're also not exclusive: build #3 as the destination, then surface it from the /skills panel with a slim "Browse mechanics guides →" entry (a few teasers linking into #3). That's discovery (in-flow) + a real library (the compendium) without duplicating the prose in the panel.
